Subdomain Takeover PoC — resources.atlas.wooga.com

Non-destructive security research conducted under the Playtika HackerOne Bug Bounty Program.

Finding

Vulnerability
Subdomain takeover via dangling GitHub Pages CNAME (CWE-350)
Affected host
resources.atlas.wooga.com
Scope
*.wooga.com (Playtika HackerOne in-scope wildcard asset)
Root cause
DNS A records point to GitHub Pages edge (185.199.108-111.153) while no GitHub repository owns the custom domain. The Wooga GitHub organization does not have wooga.com verified, allowing any third-party repo to claim the CNAME.
Proof
You are reading HTML served from a repository outside the wooga GitHub organization, via a GitHub-issued Let's Encrypt certificate for resources.atlas.wooga.com. This confirms successful hijack.

PoC deployed at
PoC contains
Static HTML only. No JavaScript runs outside this page. No cookies read/written. No external requests. No redirects. No forms. No tracking. No credential harvesting.

Remediation (in priority order)

  1. Remove the dangling A records for resources.atlas.wooga.com from the wooga.com authoritative zone.
  2. Verify the wooga.com domain in the Wooga GitHub Organization (Settings → Verified Domains) to prevent re-exploitation across all *.wooga.com subdomains.
  3. Audit CI/CD and build tooling (Atlas/Unity pipelines) for hardcoded references to resources.atlas.wooga.com and redirect them to a Wooga-controlled host.

Contact Playtika Security at security@playtika.com or via the HackerOne program to confirm this PoC should be taken down.
